Part 1 - Barbershop Problem



1. Problem Overview
  The barbershop problem is a synchronization problem where a barber
  sleeps if no customers are present, and customers either get a haircut if a
  chair is available or leave ("balk") if the waiting room is full. The goal is
  to synchronize multiple customer processes and a single barber process using
  semaphores to prevent race conditions.

2. Design Approach
The solution implements a counting semaphore and mutual exclusion approach,
taken from the textbook solution:

- Shared variables:
  - waiting: number of customers currently waiting in the waiting room.

- Semaphores:
  - mutex: binary semaphore protecting access to waiting.
  - customer: counts waiting customers; barber waits on this.
  - barber: used for customer to wait for the barber to be ready.
  - customerDone and barberDone: synchronize the end of a haircut.

Logic:

- Customer process:
  1. Get mutex to check if the waiting room is full (waiting == CHAIRS).
  2. If full, release mutex and run balk().
  3. Otherwise, increment waiting, and then release mutex.
  4. Signal customer semaphore to notify the barber.
  5. Wait on barber semaphore to occupy the barber chair.
  6. Run getHairCut().
  7. Signal customerDone and wait for barberDone to complete haircut.
  8. Decrement waiting under mutex protection.

- Barber process:
  1. Wait on customer semaphore (sleep if no customer).
  2. Signal barber semaphore to allow a customer to sit.
  3. Run cutHair().
  4. Wait for customerDone and then signal barberDone.

3. Implementation Details
- Constants:
  - CHAIRS: maximum number of waiting chairs.
  - CUSTOMERS: total number of customers to simulate.
  - STACKSIZE: size of stack for each process.

- Functions:
  - Customer_proc(void *arg): simulates a customer.
  - Barber_proc(void *arg): simulates the barber.
  - getHairCut(int id): prints which customer is getting a haircut.
  - balk(int id): prints which customer is leaving.
  - cutHair(): prints that the barber is cutting hair.

- Semaphores:
  - Created using NewSem(initialValue) from the UBC Pthreads library.
  - Operations are synchronized using P(sem) and V(sem).

- Verbosity:
  - The customer ID is printed for both haircuts and balks to track actions.

4. Testing Approach
- Implemented in Lab3.testsynchproblem.c.
- Creates the same number of customers and a barber process.
- Each customer prints its PID and ID.
- The barber prints each haircut in order.
- Validates that:
  1. Customers either get a haircut or balk correctly.
  2. No race conditions occur in waiting counter.
  3. Barber waits/sleeps if no customers are present.

5. Execution Notes
- Compile with provided Makefile:
  make clean
  make
  ./barbershop          # runs the main barbershop simulation
  ./test_barbershop     # runs the test program
- Ensure that mainp() is used as the entry point instead of main(), as 
  required by the UBC Pthreads library.

6. Limitations / Assumptions
- Program terminates when all customers have either received haircuts or 
  balked.
- Sleeping times are simulated with Sleep() to represent haircut duration.
- CHAIRS < CUSTOMERS scenario is handled by balk logic.

8. Conclusion
This solution uses semaphores to synchronize multiple customer processes with
a single barber process, ensuring mutual exclusion for shared variables and 
correct sequencing of haircuts. The test file validates that the 
implementation works as intended for the given number of customers and chairs.
